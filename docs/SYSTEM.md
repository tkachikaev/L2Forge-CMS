# Системная информация

Вкладка доступна администраторам по адресу:

```text
/admin/settings/system
```

Она предназначена для диагностики установки и подготовки безопасного отчёта для технической поддержки.

## Версия L2Forge CMS

Единственным источником версии установленной CMS является файл `VERSION` в корне проекта:

```text
0.13.7
```

Файл содержит только номер версии без префикса `v`, названия продукта и дополнительных параметров. При установке официального патча файл заменяется вместе с кодом, поэтому вкладка «Система» и подвал админки показывают новую версию автоматически.

Версия не хранится в `.env`, базе данных или `composer.json`.

## Какие сведения определяются автоматически

При каждом открытии страницы CMS заново получает:

- версию PHP и Laravel;
- версию Composer, когда команда Composer доступна веб-процессу;
- семейство операционной системы, архитектуру и PHP SAPI;
- среду Laravel и состояние режима отладки;
- драйверы кэша, сессий, очередей, почты и логов;
- часовые пояса PHP и CMS;
- драйвер, версию и состояние подключения к базе CMS;
- относительный путь и размер SQLite-базы;
- доступность служебных каталогов для реальной записи;
- состояние почтовой конфигурации;
- наличие обязательных и дополнительных расширений PHP.

Кнопка «Обновить сведения» повторяет проверки. Постоянный фоновый опрос сервера не используется.

## Проверка каталогов

Доступность каталогов определяется не только системным флагом `writable`. CMS создаёт небольшой временный файл и проверяет, что запись действительно выполнена. Удаление диагностического файла выполняется отдельно и с повторной попыткой, поэтому кратковременная блокировка файла антивирусом Windows не превращается в ложную ошибку прав записи.

Проверяются:

```text
storage/
bootstrap/cache/
public/uploads/news/
public/uploads/pages/
public/uploads/settings/
```

## Версия Composer

Composer обычно не является PHP-зависимостью приложения. CMS пытается безопасно выполнить фиксированную команду `composer --version`. Если веб-процесс не видит Composer в `PATH` или запуск системных команд запрещён, выводится:

```text
Не удалось определить
```

Это не считается ошибкой установки. Наличие Composer отдельно проверяет `doctor.ps1`.

## Безопасный отчёт

Кнопка «Скопировать отчёт» формирует текст с версиями и общими параметрами окружения. В отчёт намеренно не включаются:

- `APP_KEY`;
- пароли и токены;
- cookies и заголовки авторизации;
- SMTP-пароль;
- логины и пароли баз данных;
- полные строки подключения;
- абсолютные пути пользователя и сервера;
- содержимое `.env`.

Такой отчёт можно приложить к обращению о проблеме с установкой или обновлением.

## Скрипты установки и диагностики

`setup.ps1`, `update.ps1` и `doctor.ps1` читают номер версии из того же файла `VERSION`. Если файл отсутствует или содержит некорректное значение, установка или диагностика сообщит об ошибке сборки.

## Проверка языков

`doctor.ps1` проверяет наличие встроенных пакетов `ru` и `en`, корректность их JSON и одинаковый набор системных ключей. Дополнительный пакет может быть неполным: отсутствующие строки будут выводиться на резервном языке.

## Контекстная проверка окружения

`doctor.ps1` читает фактические значения `.env`. Для SQLite он проверяет `pdo_sqlite` и настроенный файл базы, для MySQL/MariaDB — `pdo_mysql`. При `GAME_ADAPTER=mobius` `pdo_mysql` проверяется дополнительно. SQLite-файл не считается обязательным при `DB_CONNECTION=mysql`.

Потенциально опасные production-настройки выводятся как `[WARN]`, но не превращают диагностический запуск в ложное падение. Проверяются `APP_ENV`, `APP_DEBUG`, `APP_URL`, `APP_FORCE_HTTPS`, `SESSION_SECURE_COOKIE` и `LOG_LEVEL`.

## Системный запуск Scheduler

Laravel Scheduler должен вызываться операционной системой каждую минуту. Готовые примеры для Linux cron и Windows Task Scheduler приведены в `docs/PRODUCTION.md`.
