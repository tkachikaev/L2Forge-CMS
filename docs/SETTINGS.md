# Настройки сайта

Раздел доступен по адресу:

```text
/admin/settings
```

Верхние вкладки разделяют настройки по назначению:

- **Основные** — параметры публичного сайта.
- **Игровой сервер** — игровые миры, показываемые на сайте.
- **Логин сервер** — подготовленная вкладка для будущего подключения игровых аккаунтов.
- **Регистрация** — доступность регистрации пользователей сайта и подтверждение email.
- **Почта** — SMTP, отправитель, проверка доставки и шаблоны системных писем.
- **Система** — версии CMS и окружения, состояние компонентов и безопасный отчёт для поддержки.
- **Языки** — включённые языки, язык по умолчанию и резервный язык.

## Основные настройки

В базе CMS сохраняются следующие ключи:

```text
site.name
site.description
site.logo
site.favicon
site.timezone
site.admin_email
site.footer_text
```

Значения используются стандартной публичной темой для заголовков страниц, метаописания, логотипа, favicon, текста главной страницы и подвала.

Значение подвала по умолчанию:

```text
© 2026 L2Forge-CMS
```

## Игровые серверы

Начиная с версии 0.7.2 игровые серверы хранятся в отдельной таблице `game_servers`. Можно добавлять несколько игровых миров, изменять их и удалять.

Поля одного сервера:

```text
name        — имя сервера, обязательно
rates       — рейты, необязательно
chronicle   — хроники, необязательно
mode        — режим, необязательно
sort_order  — порядок отображения
```

Правила публичного отображения:

- имя сервера показывается всегда;
- пустые рейты не создают строку «Рейты»;
- пустые хроники не создают строку «Хроники»;
- `None` в любом регистре или пустой режим скрывает строку «Режим»;
- при нескольких серверах стандартная тема выводит каждый сервер отдельной строкой;
- если все серверы удалены, серверный блок на главной странице скрывается.

При обновлении с 0.7.1 значения `server.name`, `server.rates`, `server.chronicle` и `server.mode` автоматически копируются в первую запись `game_servers`. Старые ключи остаются в `cms_settings` только для безопасной совместимости и больше не используются интерфейсом.

Поля будущего подключения к игровой базе показываются как отключённые заглушки для каждого сохранённого сервера:

```text
Адрес сервера базы данных
Порт базы данных
Название игровой базы данных
Пользователь игровой базы данных
Пароль игровой базы данных
```

Они не имеют имён формы, не отправляются браузером и не сохраняются в базе.

## Регистрация пользователей сайта

Настройки доступны по адресу:

```text
/admin/settings/registration
```

В базе CMS используются ключи:

```text
registration.enabled
registration.email_verification_required
```

Правила:

- регистрация выключена по умолчанию;
- учётная запись сайта не является игровым аккаунтом Lineage II;
- существующие пользователи могут входить, даже когда создание новых учётных записей выключено;
- обязательное подтверждение email нельзя включить вместе с регистрацией, пока почта не прошла тестовую отправку;
- при обязательном подтверждении неподтверждённый пользователь не получает доступ к `/account`.

## Почтовые настройки

Настройки доступны по адресу:

```text
/admin/settings/mail
```

В базе CMS используются ключи:

```text
mail.smtp_host
mail.smtp_port
mail.encryption
mail.username
mail.password_encrypted
mail.from_address
mail.from_name
mail.admin_email
mail.tested_at
```

Пароль SMTP хранится только в зашифрованном виде с использованием `APP_KEY`. Сохранённый пароль никогда не вставляется обратно в HTML-форму. Пустое поле пароля при повторном сохранении оставляет текущее значение без изменений.

Любое изменение SMTP-настроек сбрасывает `mail.tested_at`. После сохранения нужно отправить тестовое письмо. Только успешная проверка разрешает включить регистрацию с обязательным подтверждением email.

Во внутренних вкладках раздела находятся шаблоны подтверждения email, восстановления пароля и уведомления об успешной смене пароля. Стандартные варианты хранятся в `config/mail_templates.php`, а пользовательские переопределения — в ключах `mail.template.*` таблицы `cms_settings`. HTML-каркас формирует CMS; администратор редактирует безопасные текстовые поля, название в шапке и разрешённые переменные. Отдельная вкладка позволяет отправить одному получателю произвольное HTML-письмо после серверной проверки опасных конструкций.

Секреты приложения, CMS-базы и будущих игровых подключений по-прежнему остаются в `.env`. Исключение — SMTP-пароль, который владелец сайта вводит в админке и который хранится в зашифрованном виде.

Подробнее: `docs/USERS.md` и `docs/MAIL.md`.

## Изображения

Логотип сохраняется в:

```text
public/uploads/settings/logo/
```

Favicon сохраняется в:

```text
public/uploads/settings/favicon/
```

В базе хранятся только относительные пути. Имена файлов создаются CMS и не зависят от исходного имени пользователя.

Допустимые форматы:

- логотип: JPG, PNG, WebP, до 5 МБ;
- favicon: PNG, WebP, ICO, до 1 МБ.

SVG запрещён. При замене или удалении изображения старый файл удаляется только из разрешённого каталога настроек.

## Часовой пояс

Часовой пояс выбирается из стандартного списка PHP. После сохранения он применяется к текущей конфигурации приложения и используется для публикации новостей по расписанию и отображения дат.

Значение `APP_TIMEZONE` в `.env` остаётся безопасным значением по умолчанию, если база CMS ещё недоступна.

## Безопасность

Настройки доступны по адресу:

```text
/admin/settings/security
```

В таблице `cms_settings` используются ключи:

```text
security.admin_login_max_attempts
security.admin_login_decay_seconds
security.admin_login_ip_per_minute
security.admin_login_ip_per_hour
security.audit_retention_days
security.admin_login_retention_days
security.logs_last_cleaned_at
```

В интерфейс вынесены только безопасные диапазоны. Нельзя отключить rate limit, установить нулевой срок хранения, очистить все журналы целиком или изменить системную конфигурацию proxy и файлового логирования.

Страница также показывает количество записей в `audit_logs` и `admin_login_logs`. Предварительная проверка не изменяет базу. Ручная очистка требует текущий пароль администратора и удаляет только строки старше настроенных сроков.

Значения `.env` остаются безопасными defaults до появления переопределений в базе. Команда `php artisan l2forge:logs-clean` использует значения из панели, если параметры `--days` и `--admin-login-days` не переданы явно.


## Системная информация

Диагностическая вкладка доступна по адресу:

```text
/admin/settings/system
```

Номер установленной CMS читается из корневого файла `VERSION`. Остальные сведения и проверки формируются заново при каждом открытии страницы. Значения на этой вкладке не редактируются и не сохраняются в базе.

Подробности: `docs/SYSTEM.md`.


## Языки и переводимые поля

Начиная с 0.10.0 русская и английская версии входят в ядро. Вкладка `/admin/settings/languages` управляет установленными языками.

Для каждого включённого языка отдельно сохраняются:

```text
site.name.<locale>
site.description.<locale>
site.footer_text.<locale>
```

Название игрового сервера также редактируется во вкладках языков. Рейты, хроники, режим, часовой пояс, email и технические параметры остаются общими. Старые ключи без суффикса сохраняются для совместимости и содержат значение языка сайта по умолчанию.

Подробности: `docs/LANGUAGES.md`.
