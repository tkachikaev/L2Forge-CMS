# Настройки сайта

Раздел доступен по адресу:

```text
/admin/settings
```

Верхние вкладки разделяют настройки по назначению:

- **Основные** — параметры публичного сайта.
- **Игровой сервер** — игровые миры, показываемые на сайте.
- **Логин сервер** — подготовленная вкладка для следующего этапа.

## Основные настройки

В базе CMS сохраняются следующие ключи:

```text
site.name
site.description
site.logo
site.favicon
site.timezone
site.admin_email
site.footer_text
```

Значения используются стандартной публичной темой для заголовков страниц, метаописания, логотипа, favicon, текста главной страницы и подвала.

Значение подвала по умолчанию:

```text
© 2026 L2Forge-CMS
```

## Игровые серверы

Начиная с версии 0.7.2 игровые серверы хранятся в отдельной таблице `game_servers`. Можно добавлять несколько игровых миров, изменять их и удалять.

Поля одного сервера:

```text
name        — имя сервера, обязательно
rates       — рейты, необязательно
chronicle   — хроники, необязательно
mode        — режим, необязательно
sort_order  — порядок отображения
```

Правила публичного отображения:

- имя сервера показывается всегда;
- пустые рейты не создают строку «Рейты»;
- пустые хроники не создают строку «Хроники»;
- `None` в любом регистре или пустой режим скрывает строку «Режим»;
- при нескольких серверах стандартная тема выводит каждый сервер отдельной строкой;
- если все серверы удалены, серверный блок на главной странице скрывается.

При обновлении с 0.7.1 значения `server.name`, `server.rates`, `server.chronicle` и `server.mode` автоматически копируются в первую запись `game_servers`. Старые ключи остаются в `cms_settings` только для безопасной совместимости и больше не используются интерфейсом.

Поля будущего подключения к игровой базе показываются как отключённые заглушки для каждого сохранённого сервера:

```text
Адрес сервера базы данных
Порт базы данных
Название игровой базы данных
Пользователь игровой базы данных
Пароль игровой базы данных
```

Они не имеют имён формы, не отправляются браузером и не сохраняются в базе.

## Изображения

Логотип сохраняется в:

```text
public/uploads/settings/logo/
```

Favicon сохраняется в:

```text
public/uploads/settings/favicon/
```

В базе хранятся только относительные пути. Имена файлов создаются CMS и не зависят от исходного имени пользователя.

Допустимые форматы:

- логотип: JPG, PNG, WebP, до 5 МБ;
- favicon: PNG, WebP, ICO, до 1 МБ.

SVG запрещён. При замене или удалении изображения старый файл удаляется только из разрешённого каталога настроек.

## Часовой пояс

Часовой пояс выбирается из стандартного списка PHP. После сохранения он применяется к текущей конфигурации приложения и используется для публикации новостей по расписанию и отображения дат.

Значение `APP_TIMEZONE` в `.env` остаётся безопасным значением по умолчанию, если база CMS ещё недоступна.

## Секреты

В разделе не хранятся:

- пароли баз данных;
- `APP_KEY`;
- SMTP-пароли;
- рабочие логины подключения к игровому серверу.

Такие параметры пока остаются в `.env`. Безопасная схема подключения будет добавлена вместе с выбранным адаптером L2J Mobius.
