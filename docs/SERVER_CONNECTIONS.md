# Подключения LoginServer и GameServer

Начиная с версии 0.12.0 реквизиты внешних баз настраиваются в административной панели. Собственная база CMS остаётся отдельной и не смешивается с таблицами игрового сервера.

## LoginServer

Раздел:

```text
/admin/settings/login-server
```

Один LoginServer содержит:

```text
name
 driver
database_host
database_port
database_name
database_username
database_password
database_charset
service_host (необязательно)
service_port (по умолчанию 2106)
```

Поддерживаемые драйверы:

- `l2j_mobius` — `L2J Mobius — Interlude и новее`; таблица `accounts` обязательна, `account_data` и `accounts_ipauth` проверяются как необязательные;
- `l2j_mobius_legacy` — `L2J Mobius Legacy — C1/C4`; используется только таблица `accounts`;
- `rusacis` — заглушка: соединение с MariaDB/MySQL проверяется, но таблицы сборки пока не анализируются.

Один LoginServer может использоваться несколькими игровыми мирами. Удаление запрещено, пока на него ссылается хотя бы один GameServer или игровой аккаунт пользователя CMS. Удаление выполняется транзакционно с блокировкой записи и дополнительно защищено внешними ключами базы CMS.

## GameServer

Раздел:

```text
/admin/settings/game-server
```

Публичные параметры мира — название, рейты, хроники и режим — сохраняются независимо от подключения к базе. Поэтому существующий игровой мир продолжает отображаться, даже если база ещё не настроена.

Для подключения выбираются:

```text
LoginServer
GameServer driver
Use LoginServer database connection
```

Если используется подключение LoginServer, GameServer получает его хост, порт, имя базы, пользователя, пароль и кодировку. Это подходит для установок, где LoginServer и GameServer используют одну схему.

Если флажок снят, GameServer хранит собственные реквизиты базы. Это подходит для раздельных Login DB и Game DB.

В блоке «Дополнительные сетевые настройки» можно переопределить адрес и порт процесса. Если адрес пуст, мониторинг использует адрес базы данных. Для LoginServer порт по умолчанию `2106`, для GameServer — `7777`. Эти параметры относятся только к проверке запущенной службы и не заменяют реквизиты MySQL/MariaDB.

Поддерживаемые драйверы:

- `l2j_mobius_ct0_interlude` — рабочая проверка таблицы `characters` и дополнительных таблиц `account_gsdata`, `account_premium`;
- `rusacis` — заглушка до получения и анализа структуры базы RUSaCis.

## Проверка подключения

Кнопка «Проверить подключение» выполняет только чтение:

1. создаёт временное именованное Laravel-подключение;
2. подключается к MySQL/MariaDB с таймаутом;
3. получает версию сервера базы данных;
4. проверяет обязательные и необязательные таблицы выбранного драйвера;
5. проверяет необходимые колонки;
6. закрывает и удаляет временное подключение.

Проверка ничего не создаёт и не изменяет во внешней базе.

Для современного Mobius LoginServer (`l2j_mobius`) обязательна таблица `accounts`. `account_data` и `accounts_ipauth` отмечаются как дополнительные таблицы. Legacy-драйвер (`l2j_mobius_legacy`) для C1/C4 проверяет только `accounts` и не требует присутствия остальных таблиц.

Для Mobius CT0 Interlude GameServer обязательна таблица `characters`, включая колонку даты создания персонажа `createDate`. `account_gsdata` и `account_premium` считаются дополнительными.

Имя колонки даты не зашито в шаблон кабинета. Реестр драйверов сопоставляет колонку конкретной сборки с единым полем `created_at`: Mobius использует `createDate`, а будущий драйвер сможет указать, например, `DateCreate` или другое фактическое имя.

## Ограничение внешних запросов

Операции личного кабинета создают отдельное краткоживущее подключение. Для каждого такого подключения CMS устанавливает серверный лимит выполнения запросов:

- MySQL 5.7 и новее — `max_execution_time`;
- MariaDB — `max_statement_time`.

Значения задаются в `.env`:

```dotenv
EXTERNAL_DB_CONNECT_TIMEOUT_SECONDS=3
EXTERNAL_DB_QUERY_TIMEOUT_MS=3000
EXTERNAL_DB_CHARACTER_LIMIT=50
```

`PDO::ATTR_TIMEOUT` используется только как ограничение установления соединения. Серверный query timeout отдельно ограничивает выполнение запросов чтения. Поиск аккаунтов и персонажей выполняется прямым сравнением по индексированным полям без оборачивания колонок в `LOWER()`. Если конкретный GameServer не ответил вовремя, карточка аккаунта продолжает открываться, а данные персонажей этого мира помечаются временно недоступными.

## Хранение секретов

Пароли внешних баз:

- находятся только в собственной базе CMS;
- шифруются стандартным encrypted cast Laravel через `APP_KEY`;
- не возвращаются в HTML после сохранения;
- не сохраняются в old input сессии после ошибок валидации;
- не записываются в аудит;
- сохраняются без изменения, если поле пароля оставлено пустым при редактировании.

`APP_KEY` должен входить в защищённую резервную копию. При его утрате сохранённые пароли подключений нельзя расшифровать.

## Аудит

Фиксируются события:

```text
login_server.created
login_server.updated
login_server.deleted
login_server.connection_tested
game_server.connection_updated
game_server.connection_tested
```

Аудит содержит имя драйвера, результат подключения и совместимость схемы, но не содержит пароль.

## Текущие ограничения

Операции LoginServer с таблицей `accounts` поддерживают оба драйвера Mobius: современный `l2j_mobius` и Legacy `l2j_mobius_legacy`. Просмотр персонажей пока реализован только для GameServer `l2j_mobius_ct0_interlude`. Пока не реализованы:

- привязка существующих игровых аккаунтов;
- административное редактирование игровых данных;
- рабочие таблицы драйвера RUSaCis.
## Удаление подключений

Удаление LoginServer и GameServer выполняется через подтверждение в интерфейсе. LoginServer нельзя удалить, пока он используется игровым сервером или игровым аккаунтом пользователя. Кнопка подтверждения вызывает отдельное Livewire-действие `deleteServer` и блокируется на время выполнения запроса.


## Связанные аккаунты при удалении GameServer

Удаление GameServer не удаляет аккаунт из внешней базы LoginServer. Если у того же LoginServer остаётся другой игровой мир, CMS переназначает связанные пользовательские карточки на него. Если удалён последний мир, карточки скрываются, а поле `registration_game_server_id` очищается. Подключение нового GameServer к этому LoginServer автоматически восстанавливает скрытые карточки. Временно скрытые записи продолжают учитываться в лимите пользователя. Для последнего GameServer с активными связями CMS рассчитывает количество затронутых аккаунтов и требует подтверждение, привязанное к отпечатку этого расчёта. Серверная часть повторяет расчёт внутри транзакции; при изменении числа аккаунтов или появлении сервера-замены операция останавливается до нового подтверждения. Старый HTTP-endpoint не выполняет такое опасное удаление и направляет администратора к текущей карточке Livewire.
## Мониторинг служб и онлайна

Команда:

```powershell
php artisan kaevcms:servers-monitor
```

выполняет проверку, только если сохранённый снимок устарел согласно интервалу из панели. Для безусловного ручного обновления используется:

```powershell
php artisan kaevcms:servers-monitor --force
```

Один цикл проверки:

1. открывает краткое TCP-соединение к порту каждого LoginServer;
2. открывает краткое TCP-соединение к порту каждого GameServer;
3. для поддерживаемого GameServer-драйвера считает персонажей с `characters.online = 1`;
4. сохраняет результат и время проверки в собственной базе CMS.

После первой и второй неудачной проверки статус становится серым «Статус уточняется». Красный статус появляется после трёх последовательных неудач. Одна успешная проверка сразу возвращает зелёный статус.

Публичная главная и административный дашборд сначала читают сохранённый снимок. Если хотя бы один сервер не проверялся дольше безопасного интервала, страница запускает отдельный запрос к CMS. Только этот запрос проверяет порты и считает онлайн, после чего интерфейс обновляется без перезагрузки. Атомарная блокировка не позволяет нескольким посетителям одновременно запустить одинаковый опрос. Первый посетитель после долгого простоя видит нейтральный статус «Статус уточняется», а не старый красный результат.

Для штатного отображения статусов системный cron или Планировщик Windows не требуется. Команда зарегистрирована в Laravel Scheduler как необязательный способ продолжать проверки при полном отсутствии посетителей. На дашборде также доступна ручная кнопка «Проверить сейчас».

Интервал автоматического обновления настраивается по адресу `/admin/settings/admin-panel`. Доступны только безопасные значения: 30 секунд, 1 минута, 2 минуты и 5 минут; по умолчанию используется 1 минута. Значение сохраняется в `cms_settings` под ключом `server_monitor.refresh_interval_seconds`. Параметр `SERVER_MONITOR_REFRESH_INTERVAL_SECONDS` остаётся значением по умолчанию до первого сохранения настройки в панели. Защитная блокировка параллельных проверок не выводится в интерфейс и имеет внутренний минимум 150 секунд.


Административный дашборд показывает состояние каждого процесса независимо. На публичном сайте мир получает зелёный статус только тогда, когда отвечают его GameServer и связанный LoginServer. Онлайн считается по базе конкретного GameServer и может оставаться видимым при временной недоступности LoginServer, если снимок ещё актуален.
