# Базовые требования безопасности

- CMS и игровая база используют разные подключения.
- Пользователь MySQL для сайта получает только необходимые права.
- Тема не имеет прямого доступа к базе данных.
- Администраторы CMS, пользователи сайта и будущие игровые аккаунты разделены.
- CSRF включён глобально.
- Cookie сессии защищены настройками Laravel, `HttpOnly` включён.
- В production принудительно используется HTTPS.
- Для входа, регистрации, повторной отправки подтверждения и восстановления пароля настроен rate limit.
- Запрещено хранить игровой пароль в базе CMS или логах.

## Пользователи сайта

- Логины и email новых пользователей нормализуются в нижний регистр.
- Логин ограничен латинскими буквами, цифрами, дефисом и подчёркиванием.
- Логин и email должны быть уникальными.
- Пароли хешируются настроенным драйвером Argon2id и не сохраняются в открытом виде.
- После входа идентификатор сессии регенерируется; при выходе сессия инвалидируется и CSRF-токен обновляется.
- Подтверждение email использует временные подписанные ссылки.
- Токены восстановления пароля одноразовые, ограничены сроком действия и частотой выдачи.
- Форма восстановления возвращает одинаковое публичное сообщение для известных и неизвестных адресов.
- Пользователь сайта не получает доступа к административному guard и не является игровым аккаунтом.
- Отключённый пользователь не может войти; его remember-токен инвалидируется, а защищённая сессия завершается.
- Администратор не видит и не задаёт пароль пользователя: для смены отправляется стандартная одноразовая ссылка восстановления.

## SMTP и почта

- SMTP-пароль шифруется средствами Laravel с использованием `APP_KEY`.
- Сохранённый пароль не выводится обратно в HTML, не включается в сообщения об ошибках и не записывается в журналы.
- Пустое поле пароля при редактировании не стирает существующий секрет.
- После изменения любого SMTP-параметра предыдущий статус успешной проверки сбрасывается.
- Регистрацию с обязательным подтверждением email нельзя включить до успешной тестовой отправки.
- При потере или замене `APP_KEY` старый SMTP-пароль необходимо ввести заново.
- `APP_KEY`, доступ к CMS-базе и будущие реквизиты игровых баз остаются в `.env`.

## Новости и загрузка изображений

- HTML новости очищается на сервере по списку разрешённых тегов и атрибутов.
- Разметка из браузера никогда не считается доверенной.
- Разрешены только JPEG, PNG и WebP; SVG запрещён.
- MIME-тип, размер файла и размеры изображения проверяются до сохранения.
- Имена файлов генерируются CMS и не зависят от исходного имени.
- Встроенные изображения могут ссылаться только на каталог `/uploads/news/content/`.
- Загрузка доступна только авторизованному администратору, защищена CSRF и ограничена по частоте.
- Предпросмотр новости доступен только администратору, не сохраняет данные и запрещён для индексации и кэширования.
- Перед удалением любого файла CMS проверяет допустимый путь внутри `public/uploads/news/` и отсутствие ссылок из других новостей.
- Команда очистки удаляет только старые неподключённые JPG, PNG и WebP из каталогов обложек и содержимого.

## Основные настройки и изображения сайта

- Логотип принимает только JPEG, PNG и WebP; favicon — PNG, WebP и ICO.
- SVG запрещён, поскольку может содержать активный код.
- MIME-тип и структура изображения проверяются сервером, а имя файла генерируется CMS.
- Пути к логотипу и favicon проходят строгую нормализацию перед выводом или удалением.
- Файлы настроек удаляются только из каталогов `public/uploads/settings/logo/` и `public/uploads/settings/favicon/`.
- Поля подключения к игровой базе отключены, не имеют имён формы и не отправляются браузером.


## Журнал действий

Журнал хранит только данные, необходимые для аудита. Сервис `AuditLogger` автоматически маскирует ключи, содержащие пароли, токены, cookies, secrets и приватные ключи. Разработчики всё равно не должны передавать секреты в `details`: вместо значения фиксируется только факт изменения.

Доступ к `/admin/logs` защищён административной сессией. Записи нельзя изменять или выборочно удалять через браузер. Устаревшие записи очищаются командой `php artisan l2forge:logs-clean`.

## Администраторы

- Все административные учётные записи используют отдельный guard `admin`.
- Собственную и последнюю активную учётную запись нельзя отключить.
- Отключённая учётная запись теряет доступ при следующем запросе к панели.
- Для смены собственного пароля требуется текущий пароль.
- Администраторы не удаляются через веб-интерфейс, чтобы сохранить историю действий.
