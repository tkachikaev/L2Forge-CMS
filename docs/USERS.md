# Пользователи сайта

Начиная с версии 0.8.0 L2Forge CMS поддерживает отдельные учётные записи пользователей публичного сайта.

Это не игровые аккаунты Lineage II:

```text
Администратор CMS  — вход в /admin
Пользователь сайта — вход в /login и личный кабинет /account
Игровой аккаунт    — будет создаваться отдельным механизмом Login Server
```

## Настройки регистрации

Администратор управляет регистрацией здесь:

```text
/admin/settings/registration
```

Доступны два переключателя:

- разрешить создание новых пользователей сайта;
- требовать подтверждение email.

Регистрация по умолчанию выключена. Подтверждение email по умолчанию включено, но его нельзя активировать вместе с регистрацией, пока SMTP-настройки не прошли тестовую отправку.

Если почтовые настройки меняются после успешной проверки, статус проверки сбрасывается. При включённом подтверждении email публичная регистрация временно скрывается до новой успешной проверки.

## Публичные маршруты

```text
/register
/login
/logout
/account
/email/verify
/forgot-password
/reset-password/{token}
```

## Регистрация

Пользователь указывает:

- логин от 3 до 32 символов;
- email;
- пароль от 8 символов, содержащий буквы и цифры;
- повтор пароля.

Логин и email нормализуются в нижний регистр. Логин может содержать латинские буквы, цифры, дефис и подчёркивание.

Если подтверждение email выключено, новая учётная запись сразу считается подтверждённой. Если подтверждение включено, пользователь получает подписанную ссылку сроком на 60 минут.

## Вход

Войти можно по логину или email. После успешной авторизации сессия регенерируется. Доступ к личному кабинету при включённом подтверждении email блокируется до подтверждения адреса.

## Восстановление пароля

Форма восстановления возвращает одинаковое публичное сообщение для существующего и неизвестного email. Это не позволяет использовать форму для проверки зарегистрированных адресов.

Токены восстановления хранятся в таблице `password_reset_tokens`, действуют 60 минут и ограничены по частоте повторной отправки.

## Личный кабинет

Первая версия кабинета показывает:

- логин;
- email;
- статус подтверждения;
- дату регистрации;
- пояснение, что игровой аккаунт создаётся отдельно.
## Правила логина и пароля

Требования отображаются в административной вкладке **«Настройки → Регистрация»** и на публичной форме регистрации.

- логин: от 3 до 32 символов, латинские буквы, цифры, дефис и подчёркивание;
- пароль: не менее 8 символов, минимум одна буква и одна цифра.

Ошибки проверки этих правил выводятся на русском языке.
