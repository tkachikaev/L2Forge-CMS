# Пользователи сайта

Начиная с версии 0.8.0 L2Forge CMS поддерживает отдельные учётные записи пользователей публичного сайта.

Это не игровые аккаунты Lineage II:

```text
Администратор CMS  — вход в /admin
Пользователь сайта — вход в /login и личный кабинет /account
Игровой аккаунт    — создаётся в отдельном личном кабинете через настроенный LoginServer
```

## Настройки регистрации

Администратор управляет регистрацией здесь:

```text
/admin/settings/registration
```

Доступны два переключателя:

- разрешить создание новых пользователей сайта;
- требовать подтверждение email.

Регистрация по умолчанию выключена. Подтверждение email по умолчанию включено, но его нельзя активировать вместе с регистрацией, пока SMTP-настройки не прошли тестовую отправку.

Если почтовые настройки меняются после успешной проверки, статус проверки сбрасывается. При включённом подтверждении email публичная регистрация временно скрывается до новой успешной проверки.

## Публичные маршруты

```text
/register
/login
/logout
/account
/email/verify
/forgot-password
/reset-password/{token}
```

## Регистрация

Пользователь указывает:

- логин от 3 до 32 символов;
- email;
- пароль от 8 символов, содержащий буквы и цифры;
- повтор пароля.

Логин и email нормализуются в нижний регистр. Логин может содержать латинские буквы, цифры, дефис и подчёркивание.

Если подтверждение email выключено, новая учётная запись сразу считается подтверждённой. Если подтверждение включено, пользователь получает подписанную ссылку сроком на 60 минут.

## Вход

Войти можно по логину или email. После успешной авторизации сессия регенерируется. Доступ к личному кабинету при включённом подтверждении email блокируется до подтверждения адреса.

## Восстановление пароля

Форма восстановления возвращает одинаковое публичное сообщение для существующего и неизвестного email. Это не позволяет использовать форму для проверки зарегистрированных адресов.

Токены восстановления хранятся в таблице `password_reset_tokens`, действуют 60 минут и ограничены по частоте повторной отправки.

После успешной установки нового пароля CMS отправляет отдельное уведомление по редактируемому шаблону «Пароль изменён». Ошибка этого уведомления не отменяет уже выполненную смену пароля.

## Личный кабинет

Начиная с версии 0.13.0 `/account` является отдельной минималистичной зоной игрока. В ней доступны:

- обзор пользователя и установленного лимита;
- создание игрового аккаунта с выбором GameServer;
- список собственных игровых аккаунтов;
- карточка аккаунта и персонажи по игровым мирам;
- безопасная смена игрового пароля.

Подробности: `docs/PLAYER_ACCOUNT.md`.
## Правила логина и пароля

Требования отображаются в административном разделе **«Пользователи → Регистрация»** и на публичной форме регистрации.

- логин: от 3 до 32 символов, латинские буквы, цифры, дефис и подчёркивание;
- пароль: не менее 8 символов, минимум одна буква и одна цифра.

Ошибки проверки выводятся на выбранном языке интерфейса.

## Управление пользователями в админке

Начиная с версии 0.9.1 пользователи CMS доступны администраторам по адресу:

```text
/admin/users
```

В списке отображаются:

- логин и email;
- дата регистрации;
- дата последнего успешного входа;
- состояние учётной записи;
- состояние подтверждения email.

Доступны поиск по логину и email, фильтры по состоянию и пагинация по 50 записей.

Карточка пользователя находится по адресу `/admin/users/{id}` и содержит основные сведения и до 25 последних записей общего журнала, связанных с пользователем.

Администратор может:

- включить или отключить учётную запись;
- повторно отправить письмо подтверждения неподтверждённому пользователю;
- отправить стандартную ссылку восстановления пароля.

Администратор не может просматривать или задавать пароль пользователя. Пароль меняется только самим пользователем через одноразовую ссылку восстановления.

Отключение не удаляет запись пользователя и историю действий. Новый вход блокируется, remember-токен инвалидируется, а существующая сессия завершается. В карточке пользователя отображаются его связанные игровые аккаунты, но игровые пароли администраторам недоступны.


## Язык пользователя

В `users.locale` хранится выбранный язык сайта. Он задаётся при регистрации, обновляется при переключении языка и используется для интерфейса и системных писем.

Публичные маршруты доступны с языковым префиксом, например `/ru/account` и `/en/account`. Старые адреса без префикса сохранены для совместимости.
