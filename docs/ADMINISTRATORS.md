# Администраторы

Раздел управления администраторами находится по адресу:

```text
/admin/administrators
```

## Роли

KaevCMS использует три встроенные роли. Роль выбирается при создании или редактировании учётной записи, а под полем выбора сразу показывается краткое описание доступа.

### Владелец

Имеет полный доступ ко всей CMS, включая:

- администраторов и их роли;
- критические настройки безопасности;
- адрес административной панели;
- подключения LoginServer и GameServer;
- почту, пользователей, контент, темы и журнал действий;
- будущую установку и удаление модулей.

Последнего активного владельца нельзя отключить или понизить. Собственную роль через веб-интерфейс изменить нельзя.

### Администратор

Управляет повседневной работой CMS:

- пользователями;
- новостями и страницами;
- темами;
- LoginServer и GameServer;
- почтой;
- регистрацией и игровыми аккаунтами;
- обычными настройками, журналом и системной информацией.

Администратор не может управлять владельцами, назначать роль владельца, менять адрес панели и критические параметры безопасности. Он может создавать и изменять только администраторов и редакторов.

### Редактор

Работает только с новостями, страницами и изображениями контента. Другие рабочие разделы скрыты и недоступны даже по прямой ссылке.

Каждая роль может изменять собственное имя, email, пароль и настройки двухфакторной аутентификации.

## Обновление существующей установки

Миграция 0.23.0 назначает всем существующим администраторам роль владельца. Никто не теряет доступ после обновления. Затем владелец может вручную назначить остальным учётным записям подходящие роли. При обновлении до 0.23.3 ранее созданные модераторы автоматически становятся редакторами и должны войти повторно.

При изменении роли прежние сессии целевого администратора отзываются. На следующем запросе ему потребуется снова войти уже с новыми правами. Смена роли записывается в журнал действий.

## Список администраторов

В списке отображаются:

- имя и email для входа;
- роль;
- дата создания;
- дата последнего успешного входа;
- состояние двухфакторной аутентификации;
- состояние учётной записи;
- разрешённые текущей роли действия.

Для администратора, который ещё не входил в панель, выводится значение `Никогда`.

## Создание администратора

Нажмите **«Создать администратора»** и заполните:

- имя;
- уникальный email;
- роль;
- пароль;
- подтверждение пароля.

Email автоматически приводится к нижнему регистру и используется для входа в административную панель.

Требования к паролю:

- не менее 12 символов;
- минимум одна строчная буква;
- минимум одна заглавная буква;
- минимум одна цифра.

Пароль хранится только в виде хэша. Режим `auto` выбирает Argon2id при поддержке текущей PHP-сборкой и bcrypt в качестве совместимого резервного варианта.

Первый администратор, созданный командой `php artisan kaevcms:admin-create`, всегда получает роль владельца.

## Изменение данных и пароля

Каждый администратор может изменить собственные имя, email и пароль. Для смены собственного пароля необходимо указать текущий пароль.

Владелец может изменять данные и пароль любой другой учётной записи. Администратор может выполнять эти действия только для администраторов и редакторов. Пароли и их хэши никогда не сохраняются в журнале действий.

## Двухфакторная аутентификация

Каждый администратор подключает 2FA самостоятельно через меню пользователя **«Безопасность аккаунта»**:

```text
/admin/account/security
```

Поддерживается стандартный TOTP-код из приложения-аутентификатора и восемь одноразовых резервных кодов. Секрет хранится зашифрованным через `APP_KEY`, а резервные коды — только в виде хешей.

В списке и карточке администратора видны только статус и дата подключения. Другой администратор не может увидеть секрет, QR-код или резервные коды и не может включить 2FA за владельца аккаунта.

Владелец может аварийно сбросить 2FA другой учётной записи после ввода собственного пароля. Администратор может сделать это только для администратора или редактора. Сброс удаляет данные 2FA и отзывает активные сессии целевой учётной записи. Свою 2FA каждый администратор отключает только на странице безопасности аккаунта.

При потере доступа владелец сервера может выполнить:

```powershell
php artisan kaevcms:admin-2fa:disable admin@example.com
```

## Отключение

Физическое удаление администраторов через веб-интерфейс не предусмотрено. Учётную запись можно отключить, чтобы сохранить связь с историей действий.

Действуют ограничения:

- нельзя отключить собственную учётную запись;
- нельзя отключить последнего активного владельца;
- нельзя отключить последнюю активную административную учётную запись;
- администратор не может отключить владельца;
- отключённая учётная запись не может войти;
- уже открытая сессия отключённого администратора прекращается при следующем запросе к панели.

Отключённую учётную запись можно включить повторно без создания нового администратора.
