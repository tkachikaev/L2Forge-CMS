# KaevCMS 0.23.6

**KaevCMS** — бесплатная open-source CMS для серверов Lineage II.

До выпуска 0.20.0 проект назывался **L2Forge CMS**. Переименование не меняет данные и совместимость существующих установок.

Это ранний рабочий каркас для последовательной разработки, а не готовый публичный релиз.

## Уже есть

- Laravel 13 и PHP 8.3+.
- Отдельная база CMS и настраиваемые подключения LoginServer/GameServer к внешним базам.
- Независимые сменные темы публичного сайта `themes/<theme>` и личного кабинета `account-themes/<theme>`.
- Безопасный демонстрационный адаптер `mock`, рабочие драйверы L2J Mobius для LoginServer и CT0 Interlude GameServer, а также заглушки RUSaCis до анализа их схемы.
- Главная страница, новости, реальный статус каждого игрового мира, публичная игровая статистика, регистрация пользователей сайта и отдельный личный кабинет игрока.
- SQLite для локального запуска и тестов; для публичной установки рекомендуется MySQL или MariaDB.
- Отдельная защищённая административная панель с изменяемым адресом вида `/admin-<суффикс>` и аварийным восстановлением через Artisan.
- Три роли панели управления — владелец, администратор и редактор — с централизованной серверной проверкой доступа и режимом просмотра критических разделов.
- Единая компонентная система светлой админ-панели: сводные панели, списки, таблицы, фильтры, карточки, формы, действия и семантические статусы используют общие CSS-токены.
- Плавная Livewire-навигация с неподвижной боковой панелью, сохранением её прокрутки, предварительной загрузкой разделов и единым жизненным циклом страничного JavaScript.
- Раздельное управление установленными темами сайта и шаблонами личного кабинета.
- Раздел основных настроек сайта: название, описание, логотип, favicon, часовой пояс, email администрации и текст подвала.
- Управление несколькими игровыми серверами: компактные карточки и боковая панель с вкладками «Основное», «Статистика» и «Разное».
- Настройки регистрации пользователей сайта, SMTP-почты, трёх режимов автоматической доставки, редактируемых шаблонов и отправки произвольных HTML-писем.
- Безопасный выбор синхронной, асинхронной или сохраняемой в базе отправки автоматических писем с проверкой поддержки и контролем состояния на дашборде.
- Русский и английский интерфейс, переводы контента и поддержка дополнительных языковых пакетов.
- Подтверждение email, вход, выход и восстановление пароля.
- Журнал действий администраторов, пользователей, почты и системы.
- Управление администраторами: список, создание, изменение данных и пароля, включение и отключение.
- Персональная двухфакторная аутентификация администраторов через TOTP и одноразовые резервные коды.
- Управление пользователями CMS: поиск, фильтры, подробная карточка, активность, включение и отключение.
- Создание игровых аккаунтов Mobius из личного кабинета, включая LoginServer Legacy C1/C4 и Interlude+, настраиваемые лимиты и политика логина/пароля; временно скрытые связи продолжают учитываться в лимите.
- Единый каталог персонажей в личном кабинете: группировка по серверам и аккаунтам, плоский список «Все персонажи», поиск, фильтры, сортировка, сворачивание и постоянное скрытие ненужных групп; управление паролем остаётся на странице игрового аккаунта.
- Раздел системной информации с версиями, фактическим типом хеша паролей, диагностикой базы и trusted proxies, проверкой окружения и безопасным отчётом для поддержки.
- Trusted proxies подключаются после загрузки конфигурации Laravel, поэтому Composer, Artisan, Larastan и обновления запускаются без ранней ошибки контейнера `Target class [config] does not exist`.
- Управление новостями: список, создание, редактирование, удаление из общего списка, черновики и отложенная публикация.
- Многоязычные страницы сайта: публикация, безопасный HTML, собственные адреса, SEO, шапка, подвал и порядок навигации.
- Картинки-превью, изображения внутри новости, встроенный визуальный редактор и предпросмотр до сохранения.
- Единый серверный HTML-санитайзер с отдельными безопасными профилями для новостей, страниц и произвольных писем.
- Общий валидатор пакетов публичных тем и шаблонов личного кабинета с проверкой manifest, обязательных файлов, совместимости и безопасных путей.
- Изолированные фабрики тестовых пользователей, серверов и игровых аккаунтов для устойчивых регрессионных проверок.
- Единый слой административных операций LoginServer/GameServer; профили игровых серверов изменяются только через Livewire-панель.
- Раздельная проверка базы данных и сетевой службы: сервер считается настроенным только после подключения к совместимой схеме, а публично доступным — только при рабочей БД и службах LoginServer/GameServer.
- Реактивная смена игрового пароля в личном кабинете с ошибками без перезагрузки страницы и резервной обычной отправкой формы.
- Режим обслуживания игровых миров с оранжевым публичным статусом и отдельным сообщением для каждого включённого языка.
- Глобальное скрытие количества игроков прямо в разделе игровых серверов без отключения мониторинга и административного онлайна.

## Первый запуск в Windows

Откройте PowerShell в папке проекта:

```powershell
Set-ExecutionPolicy Bypass -Scope Process
.\setup.ps1
```

После успешной установки создайте первого администратора:

```powershell
php artisan kaevcms:admin-create
```

Запустите локальный сервер:

```powershell
.\serve.ps1
```

Откройте:

```text
Сайт:    http://127.0.0.1:8000
Админка: http://127.0.0.1:8000/admin
```

Адрес панели можно изменить в **Настройки → Панель администратора**. CMS сохраняет фиксированный префикс `admin-`, а администратор задаёт только суффикс, например `test01` для адреса `/admin-test01`. Посмотреть или восстановить адрес можно через консоль:

```powershell
php artisan kaevcms:admin-path
php artisan kaevcms:admin-path test01
php artisan kaevcms:admin-path --reset
```

`setup.ps1` выполняется при первой установке. Для обычного запуска используется только `serve.ps1`.

Релиз содержит актуальный `composer.lock`, поэтому Composer устанавливает зафиксированные версии зависимостей. Удалять этот файл перед установкой не нужно. Первое скачивание пакетов требует доступа к интернету либо заполненного локального кэша Composer.

## Диагностика

```powershell
.\doctor.ps1
```

Скрипт проверяет PHP, Composer, расширения PHP, фактически выбранный алгоритм хеширования паролей, `.env`, trusted proxies, выбранную базу CMS, служебные каталоги Laravel, встроенные языковые пакеты и реальную запись в каталоги загрузки изображений.

## Обновление проекта

После получения новой версии файлов:

```powershell
.\update.ps1
```

Скрипт сохраняет существующий `.env` и `APP_KEY`, устанавливает зависимости, выполняет миграции и тесты.


## Разделы административной панели

Настройки и служебные разделы распределены по левому меню:

- **Контент** — новости и страницы;
- **Оформление** — темы публичного сайта и личного кабинета;
- **Серверы** — игровые серверы и подключения LoginServer;
- **Пользователи** — пользователи CMS и администраторы;
- **Почта** — SMTP, доставка, системные шаблоны и произвольные письма;
- **Настройки** — единый вход в параметры сайта, панели администратора, регистрации, игровых аккаунтов, языков, безопасности и диагностики;
- **Журнал действий** — отдельный просмотр аудита административных операций;
- **Модули** — зарезервированный пункт для будущей модульной системы.

Внутри раздела **«Настройки»** используется локальная полоса вкладок. Адрес панели управления и мониторинг серверов находятся на вкладке **«Панель администратора»**, а страница **«Системная информация»** содержит только диагностику и безопасный отчёт.

Светлая административная тема использует единый визуальный слой: тонированные панели вкладок, синее активное состояние, согласованные карточки, поля и кнопки. Основные цвета, поверхности, границы, тени и радиусы вынесены в CSS-переменные `--admin-*`, чтобы будущая тёмная тема могла переопределить палитру без дублирования компонентов.

Боковая панель сохраняется между Livewire-переходами и не перерисовывается вместе со страницей. Её прокрутка остаётся на месте, раздел текущей страницы раскрывается автоматически, а ручное состояние остальных групп сохраняется в браузере.

Настройки сайта по умолчанию доступны по адресу:

```text
http://127.0.0.1:8000/admin/settings
```

Если владелец изменил суффикс панели, вместо `/admin` используется текущий адрес вида `/admin-<суффикс>`.

В разделах «Игровые серверы» и «Логин серверы» подключения отображаются компактными карточками. Создание и редактирование выполняются в боковой панели. Кнопка проверки на сохранённой карточке больше не открывает панель: зелёный или красный результат появляется прямо в карточке. При сохранении CMS также автоматически проверяет подключение в фоне интерфейса: случайные реквизиты больше не получают зелёный статус «Настроено». Игровые миры можно связывать с логин сервером и использовать общую либо отдельную базу GameServer. Для LoginServer доступны два варианта Mobius: `L2J Mobius Legacy — C1/C4` с таблицей `accounts` и `L2J Mobius — Interlude и новее` с дополнительной проверкой `account_data` и `accounts_ipauth`. Идентификатор существующего современного драйвера `l2j_mobius` сохранён. Для GameServer L2J Mobius CT0 Interlude доступны проверка схемы, чтение персонажей и публичная статистика. RUSaCis пока представлен заглушкой подключения без управления аккаунтами.

Личный кабинет `/account` использует встроенный премиальный шаблон `L2 Obsidian Luxury` и работает в постоянной Livewire-оболочке: боковая панель и верхний профиль сохраняются между страницами, а переходы выполняются через `wire:navigate` без полной перезагрузки документа. Каталог персонажей имеет режимы «По серверам» и «Все персонажи». Один сервер и аккаунт раскрываются автоматически; ненужные группы можно скрыть и затем вернуть. Плоский список всегда подписывает GameServer и игровой аккаунт, что позволяет позже безопасно добавить перевод донат-монет без изменения структуры интерфейса. Настройки отображения хранятся в `user_character_preferences`, а игровые сведения продолжают читаться напрямую через GameServer-драйвер. Подробности приведены в `docs/PLAYER_ACCOUNT.md`.

Публичный раздел `/statistics` включается отдельно для каждого GameServer. Администратор выбирает доступные рейтинги и задаёт независимый лимит от 1 до 100 для уровня, PvP, PK и игрового времени. Текущие герои и владельцы замков показываются полностью. Успешные выборки кешируются на пять минут, а кратковременная недоступность игровой базы получает отдельный защитный cooldown. Удалённые и привилегированные персонажи в рейтинги не попадают; логины аккаунтов и инфраструктурные данные не выбираются. Подробный контракт описан в `docs/GAME_STATISTICS.md`.

При удалении последнего GameServer CMS заранее показывает LoginServer и точное количество игровых аккаунтов, которые временно исчезнут из личных кабинетов. Подтверждение привязано к рассчитанным последствиям: если до нажатия кнопки число затронутых аккаунтов изменилось, администратор обязан просмотреть новый расчёт и подтвердить действие повторно. Сами аккаунты и персонажи не удаляются, а после подключения замены доступ восстанавливается автоматически.

Главная административной панели является компактным дашбордом: показывает общий онлайн, независимое состояние базы данных и службы каждого GameServer/LoginServer, время последней проверки и кнопку ручного обновления. CMS сама обновляет устаревший снимок при открытии публичной главной или административного дашборда: страница сразу показывает сохранённые данные, а отдельный защищённый запрос выполняет проверку и обновляет интерфейс без перезагрузки. Системный cron или Планировщик Windows для отображения онлайна не обязателен. Команда `kaevcms:servers-monitor` и Laravel Scheduler остаются необязательным режимом постоянного мониторинга без посетителей. Проверяются совместимость внешних схем, стандартные порты служб (`2106` и `7777`), а для Mobius CT0 Interlude считается `characters.online = 1`. Публичные карточки показывают каждый игровой мир отдельно и считают его доступным только при успешной проверке БД и службы GameServer, а также БД и службы связанного LoginServer. Для мира можно вручную включить оранжевый статус обслуживания и указать отдельное сообщение для каждого включённого языка. Время окончания при необходимости администратор пишет прямо в тексте. Количество игроков можно глобально скрыть в основных настройках сайта; административный онлайн и мониторинг при этом продолжают работать. Фиктивный лимит `5000` и шкала заполнения не используются. Интервал автоматического обновления выбирается в **Настройки → Панель администратора → Мониторинг серверов**: 30 секунд, 1 минута, 2 минуты или 5 минут. По умолчанию используется 1 минута; защитная блокировка параллельных проверок остаётся внутренним параметром CMS.

Логотип и favicon сохраняются в `public/uploads/settings`. Значение текста подвала по умолчанию:

```text
© 2026 KaevCMS
```


## Языки

Русский и английский входят в релиз и готовы сразу после установки. Управление доступно здесь:

```text
http://127.0.0.1:8000/admin/settings/languages
```

Администратор выбирает включённые языки, язык сайта по умолчанию и резервный язык. Переводятся интерфейс, новости, название и описание сайта, названия игровых серверов, сообщения об обслуживании и системные письма.

Публичные локализованные адреса:

```text
/ru
/en
/ru/news
/en/news
/ru/login
/en/login
```

Дополнительный проверенный пакет устанавливается файлами `lang/<locale>/language.php` и `lang/<locale>.json`; после этого он появляется в настройках. Загрузка языковых архивов через браузер намеренно отсутствует.

Документация: `docs/LANGUAGES.md` и `docs/development/LOCALIZATION.md`.

## Безопасность и журналы

Настройки защиты входа и сроков хранения журналов доступны здесь:

```text
http://127.0.0.1:8000/admin/settings/security
```

Администратор может изменять только безопасные диапазоны лимитов и сроков хранения. Защиту нельзя отключить через браузер. На странице показываются количество записей в `audit_logs` и `admin_login_logs`, число устаревших записей, дата последней очистки и расчёт удаления.

Кнопка ручной очистки удаляет только записи старше настроенных сроков и требует текущий пароль администратора. Полное удаление журналов через панель отсутствует. Автоматическая очистка выполняется командой `kaevcms:logs-clean`, назначенной ежедневно на 03:30; на сервере должен быть настроен запуск Laravel Scheduler.

Подробности: `docs/SECURITY.md` и `docs/AUDIT_LOG.md`.

## Пользователи сайта и почта

Регистрация настраивается здесь:

```text
http://127.0.0.1:8000/admin/settings/registration
```

Почта настраивается здесь:

```text
http://127.0.0.1:8000/admin/settings/mail
```

Во внутренних вкладках доступны готовые шаблоны подтверждения email, восстановления пароля и уведомления об успешной смене пароля. Администратор редактирует тему, название в тёмной шапке, заголовок и текст, а адаптивный HTML-каркас и реальные ссылки формирует CMS. Для каждого шаблона есть предпросмотр, тестовая отправка и восстановление стандартного варианта.

Вкладка «Отправить письмо» позволяет отправить одному получателю отдельное HTML-письмо с собственным оформлением. Полный HTML не сохраняется как шаблон, а опасные конструкции проверяются и блокируются перед отправкой.

Регистрация выключена по умолчанию. Для обязательного подтверждения email сначала сохраните SMTP-настройки и успешно отправьте тестовое письмо.

Для автоматических писем доступны три режима: синхронный, асинхронный и асинхронный с очередью в базе данных. Перед первым включением асинхронного варианта CMS выполняет реальную проверку. Режим с базой сохраняет задания в `jobs`, повторяет временно неудачные отправки и требует работающий Laravel Scheduler либо отдельный `queue:work`.

Пользователь CMS может создавать игровые аккаунты на настроенных LoginServer из отдельного личного кабинета. Ошибки смены игрового пароля выводятся рядом с полями без перезагрузки страницы; обычный HTTP-маршрут сохранён как резервный вариант. Лимит и политика логина/пароля задаются в разделе `/admin/settings/game-accounts`.
Для совместимости с игровым клиентом логины и пароли Mobius ограничены латинскими буквами и цифрами. Внешние запросы получают отдельный серверный query timeout, а ошибка чтения персонажей не блокирует страницу аккаунта целиком.

Управление пользователями CMS доступно здесь:

```text
http://127.0.0.1:8000/admin/users
```

В разделе доступны поиск и фильтры, подробная карточка, последние события журнала, включение и отключение учётной записи, повторная отправка подтверждения email и отправка ссылки восстановления пароля. Пароли и служебные токены администраторам не показываются.

Публичные страницы:

```text
/register
/login
/account
/forgot-password
```

Подробности: `docs/USERS.md`, `docs/PLAYER_ACCOUNT.md` и `docs/MAIL.md`.

## Администраторы

Управление учётными записями панели:

```text
http://127.0.0.1:8000/admin/administrators
```

Для учётных записей панели доступны роли **Владелец**, **Администратор** и **Редактор**. Владелец имеет полный доступ; администратор управляет рабочими разделами без доступа к владельцам и критическим настройкам; редактор работает только с новостями и страницами. Существующие администраторы после обновления получают роль владельца, поэтому доступ не теряется. Ранее созданные модераторы при обновлении до 0.23.3 автоматически становятся редакторами. Последнего активного владельца нельзя отключить или понизить.

Персональная защита текущего администратора находится в меню пользователя:

```text
/admin/account/security
```

Каждый администратор самостоятельно подключает TOTP-приложение, подтверждает шестизначный код и сохраняет восемь одноразовых резервных кодов. После обновления до 0.11.12 существующие административные сессии один раз потребуют повторного входа. В списке администраторов виден статус 2FA. Для чужой учётной записи доступен только аварийный сброс с подтверждением пароля инициатора; секрет и резервные коды никому не показываются.

Консольный аварийный сброс:

```powershell
php artisan kaevcms:admin-2fa:disable admin@example.com
```

Документация: `docs/ADMINISTRATORS.md` и `docs/ADMIN_AUTH.md`.

## Журнал действий

Журнал доступен по адресу:

```text
http://127.0.0.1:8000/admin/logs
```

Он фиксирует важные действия администраторов и пользователей, отправку почты и системные операции. Пароли, токены, cookies и другие секреты автоматически скрываются.

Общий журнал хранится 90 дней, журнал входов администраторов — 30 дней. Одна команда проверяет и очищает обе таблицы:

```powershell
php artisan kaevcms:logs-clean --dry-run
php artisan kaevcms:logs-clean
```

Сроки можно переопределить через `AUDIT_LOG_RETENTION_DAYS` и `ADMIN_LOGIN_LOG_RETENTION_DAYS` либо разовыми параметрами `--days` и `--admin-login-days`.

Форма `/admin/login` защищена отдельными лимитами по IP за минуту и за час, а также лимитом по связке email и IP. Запросы, остановленные IP-лимитером, не создают записи в базе. Попытки входа с неизвестным email сохраняются только в специализированном журнале входов и не дублируются в `audit_logs`.

Технический Laravel-log по умолчанию использует ежедневную ротацию и хранит 14 файлов. Настройки: `LOG_STACK=daily` и `LOG_DAILY_DAYS=14`.

На production очистка журналов и обработка почтовой очереди в базе данных зарегистрированы в Laravel Scheduler и требуют системного запуска `php artisan schedule:run`. Мониторинг LoginServer/GameServer работает и без Scheduler: устаревшие данные автоматически обновляются при посещении страницы со статусами. Синхронный и обычный асинхронный режимы почты также работают без Scheduler.

Документация: `docs/AUDIT_LOG.md` и `docs/development/AUDIT_LOGGING.md`.

## Страницы сайта

Управление информационными страницами:

```text
http://127.0.0.1:8000/admin/pages
```

Страницы поддерживают все включённые языки, собственный адрес для каждой локали, черновики, публикацию, безопасный визуальный редактор, изображения, SEO-заголовок и SEO-описание. Опубликованную страницу можно добавить в шапку или подвал сайта и задать порядок отображения.

Старые изображения, загруженные в редактор, но не использованные ни в одной странице, можно проверить и удалить:

```powershell
php artisan kaevcms:page-media-clean --dry-run
php artisan kaevcms:page-media-clean
```

Публичные адреса:

```text
/pages/server-rules
/ru/pages/pravila-servera
/en/pages/server-rules
```

Нелокализованный адрес перенаправляет на фактический перевод. Неверный slug исправляется постоянным редиректом, а при отсутствии перевода для выбранного языка посетитель перенаправляется на резервную локаль. Страница публикует `canonical`, `hreflang` и `x-default` только для реально существующих переводов.

Новый языковой пакет автоматически появляется в редакторе страниц без изменения схемы базы.

Документация: `docs/PAGES.md`.

## Темы

Публичные темы находятся в `themes/<slug>`, а их ресурсы — в `public/themes/<slug>`.

Шаблоны личного кабинета находятся в `account-themes/<slug>`, а их ресурсы — в `public/account-themes/<slug>`. Они активируются независимо от публичной темы и не содержат бизнес-логику аккаунтов или персонажей.

Управление:

```text
Публичные темы:            http://127.0.0.1:8000/admin/themes
Шаблоны личного кабинета:  http://127.0.0.1:8000/admin/account-themes
```

Перед активацией CMS проверяет манифест, обязательные представления, публичные CSS/JS и совместимость с текущей версией. Загрузка ZIP пока намеренно отключена до реализации безопасной проверки архивов.

## Контроль качества

Для разработки доступны единые команды:

```powershell
composer lint
composer analyse
composer test
composer quality
```

`composer lint` проверяет оформление через Laravel Pint. `composer analyse` запускает Larastan/PHPStan на уровне 5 для каталогов `app`, `routes` и `database`. `composer quality` последовательно выполняет проверку форматирования, статический анализ и автоматические тесты.

Все административные `FormRequest` наследуются от общего `AdminFormRequest` и явно требуют авторизованную учётную запись guard `admin`, даже если маршрут уже защищён middleware.

## Важно

`GAME_ADAPTER=mock` по-прежнему используется демонстрационным рейтингом по умолчанию. Публичный статус и онлайн больше не берутся из mock-адаптера: они читаются из сохранённых результатов фонового мониторинга настроенных LoginServer/GameServer. Игровые аккаунты личного кабинета работают независимо через драйверы внешних баз.

Фоновое изображение стандартной темы используется как макет. Перед публичным релизом его нужно заменить на собственный или лицензированный арт.
